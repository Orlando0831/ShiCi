# 把「拾词」装进 iPhone（随时随地、离线可用）

拾词现在是 **离线优先 (offline-first)**：所有词表与复习记录都**先存在手机本机**，
**离线、外出、电脑没开机时都能照常打开使用**。登录账号后，联网时会自动**云端同步**，
多设备登录同一账号即可共享。界面已针对 **iPhone 16 Pro**（灵动岛、底部安全区）完全适配。

---

## 怎样才能「随时随地（含移动网络）」打开？

iPhone 要在**电脑关机、只连移动网络**时还能打开 App，关键是 App 本身（界面外壳）要先
被手机缓存下来——这需要从一个 **HTTPS 网址**安装过一次（iOS 的离线缓存只在 HTTPS / localhost 下生效）。
所以有两种用法，按你的需要二选一：

### 方式一 · 随时随地（推荐）：部署到免费云端（一次配置，永久可用）
把 `ShiCi` 文件夹部署到一台带 HTTPS 的免费服务器，手机用它的网址安装到主屏幕，
之后**无论在哪、电脑开不开机**都能打开（外壳已缓存、数据在本机），联网时自动同步。

以 **Render.com**（免费）为例：
1. 把 `ShiCi` 文件夹推到一个 GitHub 仓库。
2. Render → New → **Web Service** → 选该仓库；填：
   - **Build Command**（必填，不能留空）：`pip install -r requirements.txt`
     （`ShiCi` 里已附 `requirements.txt`；本项目零依赖，此命令会顺利通过、不装任何东西）
   - **Start Command**：`python server.py`
   - **Environment Variables**：加一个 `HOST` = `0.0.0.0`（端口 Render 会自动用 `PORT` 注入）
3. 部署成功后得到一个网址，如 `https://shici-xxxx.onrender.com`。
4. iPhone 用 **Safari** 打开该网址 → 注册/登录 →「**分享 → 添加到主屏幕**」。
   - 之后点主屏图标即**全屏运行、可离线打开**；数据随账号在多设备间同步。

> 免费版服务器闲置会休眠（首次唤醒慢几十秒），但因为外壳已缓存、数据在本机，**离线照样秒开**，
> 联网后自动补传同步，不影响使用。

### 方式二 · 在家局域网（无需部署，但需电脑开机且同一 Wi‑Fi）
1. 电脑双击 `start_backend_lan.bat`，记下窗口里的 `LAN: http://192.168.x.x:8000/`。
2. iPhone 连**同一 Wi‑Fi**，Safari 打开该地址 → 注册/登录 →「分享 → 添加到主屏幕」。

> 注意：局域网是 `http`（非 HTTPS），iOS 不会缓存外壳，所以**这种方式离开 Wi‑Fi / 电脑关机后无法打开**。
> 想真正随时随地，请用**方式一**。

## 数据怎么搬到云端 / 备份

- 在任意一端（电脑或手机）打开「导入 / 导出」→ 导出 **JSON 备份**，再到另一端「导入」即可。
- 或者：两端登录**同一账号**，会自动通过云端合并同步。
- 同步状态：点右上角头像可见「已与云端同步 / 离线 · 已存本机 / 仅保存在本机」。

## 离线行为说明

- **未登录**也能完整使用（积累、复习、导入导出、20000 上限），数据只存本机。
- **已登录但离线**：照常使用，改动先存本机，**联网或切回 App 时自动同步**。
- 卸载 App 或清除 Safari 数据会清掉本机数据；重要数据建议登录云同步或定期导出 JSON 备份。
